    }

    public class StrategyFactory {
        private Map<Integer, Strategy> map;
    
        public StrategyFactory() {
    
            List<Strategy> strategies = new ArrayList<>();
    
            strategies.add(new Level1Strategy());
            strategies.add(new Level2Strategy());
    
            map = strategies.stream().collect(Collectors.toMap(Strategy::getType, strategy -> strategy));
    
        }
       /*
       静态内部类单例，单例模式实现的一种,在构造方法的时候，初始化好需要的 Strategy，并把 list 转化为 map。这里 转化就是“灵魂”所在
       */
        public static class Holder {
            public static StrategyFactory instance = new StrategyFactory();
        }
    
        public static StrategyFactory getInstance() {
            return Holder.instance;
        }
    
        public Strategy get(Integer type) {
            return map.get(type);
        }
    }

 //优化后
    private static double getResult2(long money, int type) {
        if (money < 1000) {
            return money;
        }

        Strategy strategy = StrategyFactory.getInstance().get(type);

        if (strategy == null){
            throw new IllegalArgumentException("please input right type");
        }

        return strategy.compute(money);
    }
}

5.总结：采用工厂+策略+静态内部类单列 优雅处理了复杂if-else的代码。 等有空的时候，尝试应用到实际业务中优化代码。今年目标是深入学习各类框架源码+策略模式，改变自己写代码low的现状，毕竟谁也不想做一个只会cv的码畜。
